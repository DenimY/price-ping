## Price Ping 기획서 (초안)

### 1. 서비스 개요

- **서비스 이름**: Price Ping  
- **한 줄 소개**: 관심 상품의 **가격 변동을 자동으로 추적**하고, 설정한 조건에 맞으면 **앱/카카오톡으로 바로 알려주는 알림 서비스**
- **초기 타겟 마켓**
  - 커머스: 네이버 스토어 (향후 쿠팡, G마켓 등 확장)
  - 사용자는 상품 URL을 등록해 즐겨찾기처럼 관리

### 2. 문제 정의 & 제공 가치

- **사용자 문제**
  - 사고 싶은 상품이 있어도, 매일 가격을 확인하기 번거로움
  - 특정 가격 이하로 떨어지면 사고 싶은데, 그 시점을 놓치기 쉬움
  - 여러 쇼핑몰 가격을 비교·추적하는 것이 번거로움
- **제공 가치**
  - 사용자는 **원하는 가격/조건만 정해두고 잊어버려도 됨**
  - Price Ping이 대신 가격을 추적하고, 조건 만족 시 **즉시 알림**
  - 향후 여러 몰을 묶어서 **한 곳에서 가격 히스토리/변동 그래프** 확인 가능

### 3. 타겟 사용자 & 페르소나

- **타겟**
  - 온라인 쇼핑을 자주 하고, **특가/세일**에 민감한 20–40대
  - 전자제품, 가전, 고가 상품 등 **가격 변동 폭이 큰 카테고리** 구매자
- **예시 페르소나**
  - 34세, 직장인, 전자제품/생활가전 구매 전 **최저가 트래킹**을 자주 하는 사람
  - 네이버 쇼핑을 주로 사용하고, 여러 상품을 장바구니/찜에 넣어두고 가격을 지켜보는 유형

### 4. 핵심 기능 정의

#### 4.1 MVP 범위 (최소 기능)

- **회원 / 인증**
  - **이메일 기반 계정(이메일 + 비밀번호)**만 사용 (소셜 로그인은 이후 확장)
- **상품 관리**
  - 네이버 스마트스토어 / 네이버플러스 스토어 상품 URL 등록
  - 등록 시 기본 정보 수집: 제목, 이미지, 현재 가격 등
  - 내 관심 상품 리스트 조회 / 삭제
- **가격 모니터링**
  - 일정 주기로 상품 가격을 조회 (예: 10–30분/1시간 단위, 초기에는 여유 있게)
  - 가격 변동 내역 저장 (최신 가격, 체크 시각)
- **알림 설정 (Alert Rules)**
  - 상품별로 **목표 가격(Target Price)** 설정
  - 가격이 **목표 이하로 떨어지면** 알림
  - (간단 버전) 목표 가격 1개만 먼저 지원
- **알림 발송**
  - 웹/PWA 기준 브라우저 푸시 알림
  - 카카오톡 알림(비즈메시지/채널 API 연동) – 최소 1개 템플릿으로 시작

#### 4.2 이후 확장 기능

- 가격 변동 그래프(가격 히스토리 차트)
- 알림 조건 확장
  - 일정 % 이상 하락 시 알림
  - 일정 % 이상 상승 시 알림
- 다른 쇼핑몰 지원: 쿠팡, G마켓, 11번가 등
- 카테고리/태그 관리, 정렬/필터

### 4.3 인증 및 회원 정책

- **지원 로그인 방식 (초기)**
  - **이메일 + 비밀번호** (Supabase Email/Password) 만 지원
    - 회원가입 시:
      - 이메일, 비밀번호 입력
      - **이메일 검증(Verification 메일)** 필수: 메일 내 링크 클릭 시 계정 활성화
    - 비밀번호 재설정:
      - “비밀번호 찾기”에서 이메일 입력 → 재설정 링크 발송
      - 링크를 통해 진입한 페이지에서 새 비밀번호 입력
  - **소셜 로그인 (향후 확장)**
    - 초기 버전에는 포함하지 않음.
    - 추후 필요 시 **구글, 카카오** 순으로 도입을 검토하며,
      - 구글: Supabase Google Provider
      - 카카오: Supabase + 별도 OAuth 연동 후 `kakao_links` 매핑
    - 네이버 로그인은 약관/심사 및 운영 부담을 검토 후 별도 결정.

- **정책**
  - 이메일 로그인 계정의 경우, **이메일 검증 완료 전까지 주요 기능(상품 등록/알림 설정 등)은 제한**.
  - 사용자 역할은 2종으로 구분:
    - **일반 유저 (`user`)**: 기본 사용자 권한
    - **관리자 유저 (`admin`)**: 운영/관리 목적 권한
  - 하나의 이메일 기준:
    - 동일 이메일로 이메일 로그인/구글/카카오 계정을 **가능한 한 통합**(Supabase Auth 설정에 따름).
  - 탈퇴 시:
    - `auth.users` 삭제 → 연관 프로필/데이터(`profiles`, `favorites`, `alert_rules` 등)는 DB 정책에 따라 연쇄 삭제 또는 익명화 (초기에는 연쇄 삭제).

- **관리자 정책**
  - 초기 버전에서는 관리자 계정도 일반 로그인 플로우(이메일 로그인)를 사용
  - 관리자 여부는 `profiles.role = 'admin'`으로 구분
  - 관리자 전용 화면/API는 추후 추가하되, DB 스키마에는 역할 구분을 먼저 반영

### 5. 사용자 시나리오 (User Flow)

1. **회원가입 & 로그인**
   - 사용자는 **이메일 + 비밀번호**로 회원가입
   - 가입 후 이메일 검증 메일 수신 → 링크 클릭 시 계정 활성
   - 비밀번호 재설정:
     - 로그인 화면의 “비밀번호 찾기” → 이메일 입력 → 재설정 링크를 통해 새 비밀번호 설정
  - 로그인 후 대시보드(내 관심 상품 리스트)로 진입
2. **상품 등록 플로우**
  - 사용자는 네이버 스마트스토어 또는 네이버플러스 스토어에서 상품 URL 복사
  - Price Ping에서 “상품 추가” 버튼 클릭 → URL 붙여넣기
  - 서버가 URL을 파싱하여
    - 상품 제목, 이미지, 현재 가격을 가져와 저장
  - 사용자는 이 상품에 대한 **목표 가격**을 입력 후 저장
3. **가격 모니터링 & 알림**
  - 백엔드(또는 Supabase Edge Function)가 스케줄에 따라 가격 체크
  - 가격이 목표 가격 이하인 경우:
    - `price_history`에 기록
    - `notifications`에 알림 레코드 생성
    - 푸시/카카오톡으로 알림 전송
4. **사용자 피드백**
  - 알림을 받은 사용자는 상품 상세(원본 상품 링크)로 이동
  - 이후 필요시 알림 조건 수정 또는 비활성화

### 6. 화면 설계 (1차 목록)

- **웹 / PWA**
  - 로그인/회원가입 페이지
  - 대시보드(내 관심 상품 리스트)
    - 상품명, 현재 가격, 목표 가격, 상태(조건 만족 여부)
  - 상품 추가 페이지/모달
    - 상품 URL 입력
    - 기본 정보 미리보기(제목, 이미지, 현재 가격)
    - 목표 가격 입력
  - 상품 상세/설정 페이지
    - 가격 히스토리(텍스트 리스트 → 이후 그래프)
    - 알림 조건(목표 가격, on/off)
  - 설정 페이지
    - 푸시 알림 허용 상태
    - 카카오 연동 상태

### 7. 기술 스택 개요 (요약)

- **Frontend**
  - Next.js (React, TypeScript)
  - PWA 구성 (설치형 사용 경험, 푸시 알림)
- **Backend / Infra**
  - Supabase (Postgres, Auth, RLS, Edge Functions)
  - 가격 크롤링 & 알림 로직을 Edge Function 또는 별도 워커로 구현
- **크롤링**
  - Edge Function에서
    - `axios` + `cheerio` (가능한 경우)
    - 필요 시 `Puppeteer`/`Playwright`로 동적 컨텐츠 처리
- **알림**
  - 웹 푸시 (브라우저 Push API + Service Worker)
  - 카카오톡 비즈메시지/채널 API

### 8. 스케줄링 & 인프라 전략

- **스케줄링 방식**
  - 1차 목표: **Supabase Scheduled Functions(크론)**를 사용해 `price-checker` Edge Function을 정기 실행.
    - 예: 1시간 주기 `cron: "0 * * * *"` 등으로 호출.
  - Supabase 크론 한계를 초과하거나 더 세밀한 제어가 필요해질 경우,
    - **외부 스케줄러(Cloudflare Workers, GitHub Actions, 별도 서버의 cron 등)**가 HTTP로 Edge Function을 호출하는 구조로 확장 가능.

- **실패/재시도 기본 정책**
  - 가격 체크 요청 단위로:
    - 네트워크/타임아웃/일시적 오류 시 **소량(예: 최대 2회)** 재시도.
    - 429/403 등 차단 관련 응답 시:
      - 재시도 대신 해당 상품의 **다음 체크 시각을 크게 늦추고**, 에러 로그에 기록.
  - Edge Function 전체 실행 실패 시:
    - Supabase 로그를 통해 원인 확인.
    - 추후 운영 단계에서 외부 모니터링/알림(Slack, 이메일 등) 연동을 고려.

- **운영/모니터링 방향**
  - 최소한으로:
    - `products`/`price_history`에 **마지막 성공 체크 시각**을 저장하여, 대시보드에서 “갱신 시각” 표시.
    - 연속된 실패(예: 동일 상품 3회 연속 실패) 시 별도 상태 플래그 또는 로그 테이블에 기록.
  - 향후:
    - 외부 APM/로깅(Sentry 등) 연동하여 Edge Function 에러 모니터링.
    - 실패율이 특정 임계치 이상일 때 운영자 알림 기능 도입.

### 9. 크롤링 리스크 & 운영 정책 (네이버 스토어)

- **법적/약관 및 로봇 정책 리스크**
  - 네이버 서비스(쇼핑/스마트스토어 등)는 이용약관 및 robots 정책에 따라 **자동화/상업적 크롤링이 제한될 수 있음**.
  - 본 서비스는 초기에는 **개인용/PoC 수준**에서만 크롤링을 활용하고, 상업 서비스 단계에서는
    - 네이버 공식 API 또는 제휴/파트너십 검토를 우선순위로 함.
  - 약관/정책 위반 소지가 있을 경우 기능 축소 또는 구조 변경을 고려.

- **기술적 차단(UA/쿠키/속도/패턴) 리스크**
  - 동일 IP·패턴으로 반복 요청 시 **차단(403/429, 캡차 등)** 가능성이 있음.
  - 대응 원칙:
    - **요청 주기 완화**: 초기에는 상품당 최소 1~3시간 이상 간격으로 체크.
    - 불필요한 헤더/쿠키 의존 최소화, UA는 고정/과도한 스푸핑은 지양.
    - 429/403 등 응답 수신 시 **지수 백오프 + 재시도 제한** 적용.

- **DOM 구조 변경/파싱 실패 리스크**
  - 가격 정보의 DOM 구조가 변경되면 파싱 로직이 깨질 수 있음.
  - 대응 원칙:
    - 가격 파싱 실패 시 `price_history`/`products`에 **실패 상태를 명시적으로 기록**하고, 잘못된 가격을 저장하지 않음.
    - 셀렉터를 하나만 쓰지 않고 **우선순위 리스트(fallback)** 로 관리.
    - 일정 기간 동안 파싱 실패율이 임계치(예: 10% 이상) 넘으면 운영 알림(추후 도입) 및 수동 점검.

- **주기/부하(QPS) 및 차단 리스크**
  - 유저/상품 수가 증가할수록 요청 수가 기하급수적으로 늘어날 수 있음.
  - 대응 원칙:
    - 유저당 **최대 관심 상품 수 제한**(예: 20개) 설정.
    - 글로벌 크롤링 배치의 주기를 **보수적으로 시작**(예: 1~3시간)하고, 운영 데이터를 보며 조정.
    - Edge Function/워커에서 배치당 처리 상품 수 상한 및 **요청 간 간격(예: 500–1000ms)** 삽입.

- **서비스 품질/기대치 관리**
  - “실시간” 가격 추적이 아닌 **“정기적/주기적 점검”**임을 사용자 문구에서 명확히 표현.
  - 알림 도착에 일정 지연(예: 최대 수 시간)이 발생할 수 있음을 FAQ/설명에 포함.
  - 크롤링이 여러 번 실패한 경우, UI에서 “일시적으로 가격 정보를 가져오지 못하고 있습니다” 등의 상태 표기.

### 10. 마일스톤 (초기 버전)

- **M1 – 기초 세팅**
  - Supabase 프로젝트 생성, 테이블/정책(RLS) 적용
  - Next.js + Supabase 연동, 기본 인증 플로우 구축
- **M2 – 상품 등록 & 리스트**

- **M3 – 목표 가격 알림**
  - 목표 가격 설정 UI/로직

  - 웹 푸시 알림 발송
- **M4 – 카카오 알림 & 정리**
  - 카카오 연동
  - 알림 로그 화면
