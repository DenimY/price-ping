## 와이어프레임 문서 (텍스트 초안)

> Figma 등 디자인 툴 작업 전에, 화면 구성을 텍스트로 정리한 초안입니다.  
> 실제 UI 작업 시 이 구조를 기반으로 컴포넌트/레이아웃을 설계합니다.

---

### 1. 공통 레이아웃

- **헤더**
  - 좌측: 로고 또는 서비스명 `Price Ping`
  - 우측:
    - 프로필 아이콘 / 이메일
    - 설정(톱니바퀴) 아이콘
    - 로그아웃 버튼

- **메인 영역**
  - 좌측 상단: 현재 페이지 제목
  - 우측 상단: 주요 액션 버튼(예: “상품 추가”)
  - 본문: 리스트/폼/차트 등 페이지별 컨텐츠

---

### 2. 로그인 / 회원가입 페이지

- **페이지 제목**
  - `로그인` / `회원가입`

- **레이아웃**
  - 중앙 정렬 카드 형태
  - 상단: 서비스 로고 + 간단한 한 줄 설명
  - 중단: 폼 영역
    - 이메일 입력
    - 비밀번호 입력
    - (선택) 비밀번호 확인 입력 (회원가입 시)
    - 제출 버튼
  - 하단:
    - “아직 계정이 없으신가요? → 회원가입”
    - “이미 계정이 있으신가요? → 로그인”

---

### 3. 대시보드 (내 관심 상품 리스트)

- **페이지 제목**
  - `내 관심 상품`

- **헤더 영역**
  - 좌: 제목(`내 관심 상품`)
  - 우: 버튼
    - `+ 상품 추가` 버튼

- **리스트 영역**
  - 각 상품은 카드 또는 테이블 행 형태
  - 항목 내용:
    - 썸네일 이미지
    - 상품명 (네이버 스토어 링크로 연결)
    - 현재 가격
    - 목표 가격 (있다면)
    - 알림 상태 (ON/OFF 토글)
    - 마지막 가격 체크 시각
    - 우측 액션:
      - `상세` 버튼 (상품 상세 페이지로 이동)
      - `삭제` 버튼 (즐겨찾기 제거)

- **빈 상태(Empty State)**
  - “아직 등록된 관심 상품이 없습니다.”
  - `+ 상품 추가` 버튼 강조

---

### 4. 상품 추가 모달 / 페이지

- **진입 경로**
  - 대시보드의 `+ 상품 추가` 버튼 클릭 시

- **구성**
  - 상단: 제목 `상품 추가`
  - 입력 필드:
    - `상품 URL` 입력 필드
      - placeholder: “네이버 스토어 상품 URL을 붙여넣어주세요”
    - `상품 정보 불러오기` 버튼
  - 정보 미리보기 영역 (URL 파싱 후 표시)
    - 썸네일 이미지
    - 상품명
    - 현재 가격
  - 알림 설정 영역
    - `목표 가격` 입력 필드
    - 알림 ON/OFF 체크박스 또는 스위치
  - 하단 버튼
    - 좌: 취소
    - 우: 저장(등록)

---

### 5. 상품 상세 / 설정 페이지

- **페이지 제목**
  - 상품명(또는 축약) + “상세”

- **상단 정보 영역**
  - 썸네일 이미지
  - 상품명
  - 쇼핑몰 이름 (네이버 스토어 등)
  - 현재 가격
  - 상품 원본 링크 버튼 (`네이버에서 보기`)

- **중단: 알림 설정 카드**
  - `알림 상태` 토글 (ON/OFF)
  - `목표 가격` 입력
  - (확장) 알림 타입 선택 드롭다운 (예: “목표 가격”, “% 하락”)
  - 저장 버튼

- **하단: 가격 히스토리 영역**
  - 초기:
    - 최근 N개의 가격 변동을 리스트로 나열
      - 날짜/시간, 가격
  - 이후:
    - 라인 차트(가격 변동 그래프) 표시 영역

---

### 6. 설정 페이지

- **페이지 제목**
  - `설정`

- **섹션 1: 계정 정보**
  - 이메일
  - 닉네임
  - 가입일

- **섹션 2: 알림 설정**
  - 웹 푸시 알림 상태 (허용/차단) 텍스트 및 버튼
    - “브라우저 알림 허용” 버튼
  - 카카오 연동 상태
    - 연동됨 / 미연동 상태 표시
    - `카카오 연동하기` / `연동 해제` 버튼

- **섹션 3: 기타**
  - 서비스 소개 / 피드백 링크

---

### 7. 모바일(PWA) 고려 사항

- **하단 내비게이션 바 (모바일 시)**
  - 아이콘 + 텍스트:
    - `홈` (대시보드)
    - `알림` (알림 로그)
    - `설정`

- **반응형 레이아웃**
  - 데스크톱: 좌측 사이드바 + 상단 헤더 구조도 고려 가능
  - 모바일: 상단 헤더 + 하단 탭 구조

---

### 8. 알림 로그 페이지 (선택)

- **페이지 제목**
  - `알림 내역`

- **리스트 항목**
  - 알림 제목
  - 메시지 요약
  - 발송 채널 (웹 푸시 / 카카오)
  - 발송 시각
  - 클릭 시: 해당 상품 상세 페이지로 이동
